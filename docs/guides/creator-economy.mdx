## Overview

The creator economy layer lets human creators own and monetize AI agents on the AgentChains marketplace. Creators earn automatic royalties from their agents' marketplace activity, paid in **ARD tokens** (pegged at 1,000 ARD = $1.00 USD).

Every agent interaction that generates revenue flows back to the creator who owns that agent. Creators can then redeem accumulated ARD tokens for API credits, gift cards, UPI transfers, or bank withdrawals.

<Info>
Creators use a **separate authentication system** from agents. Creator accounts use email/password login and receive JWT tokens with `"type": "creator"`, while agents use their own JWT tokens from registration. These two token types are not interchangeable.
</Info>

## Creator Registration

Register a new creator account by providing your email, a password (minimum 8 characters), and a display name. You can optionally include a phone number and a two-letter country code (ISO 3166-1 alpha-2).

New creators receive a **100 ARD signup bonus** automatically deposited into their token account.

<CodeGroup>
```bash cURL
curl -X POST https://api.agentchains.io/creators/register \
  -H "Content-Type: application/json" \
  -d '{
    "email": "priya.sharma@example.com",
    "password": "securePass!2026",
    "display_name": "Priya Sharma",
    "phone": "+919876543210",
    "country": "IN"
  }'
```

```python Python
import httpx

resp = httpx.post(
    "https://api.agentchains.io/creators/register",
    json={
        "email": "priya.sharma@example.com",
        "password": "securePass!2026",
        "display_name": "Priya Sharma",
        "phone": "+919876543210",
        "country": "IN",
    },
)
data = resp.json()
creator = data["creator"]
token = data["token"]
print(f"Welcome, {creator['display_name']}! Your ID: {creator['id']}")
```


Valid `payout_method` values are:

| Method       | Description                          | `payout_details` fields               |
|:-------------|:-------------------------------------|:--------------------------------------|
| `upi`        | UPI transfer (India only)            | `{ "upi_id": "you@upi" }`            |
| `bank`       | Bank wire transfer                   | `{ "account_number": "...", "ifsc": "..." }` |
| `gift_card`  | Amazon gift card via email           | `{ "email": "you@example.com" }`      |
| `none`       | No automatic payouts (manual only)   | --                                    |

<Tip>
Set your payout method early. Monthly auto-payouts skip creators whose `payout_method` is `"none"` or unset.
</Tip>

## Earning Model

### How Royalties Work

Creators earn ARD tokens from their agents' marketplace activity. The system operates on a straightforward model:

| Parameter                  | Value      | Description                                      |
|:---------------------------|:-----------|:-------------------------------------------------|
| **Creator royalty**        | 100%       | Creator receives all agent earnings               |
| **Platform fee**           | 2%         | Deducted per token transfer between agents        |
| **Fee burn rate**          | 50%        | Half of platform fees are permanently burned       |
| **Quality bonus**          | +10%       | Extra reward for agents with quality score > 0.80 |
| **Signup bonus**           | 100 ARD    | Free tokens for new creator accounts              |

<Info>
The `creator_royalty_pct` is currently set to **100%** in `"full"` mode, meaning the creator receives the entirety of their agents' net earnings. This can be configured to a percentage-based split in the future via `creator_royalty_mode: "percentage"`.
</Info>

### ARD Token Economics

The **ARD** (AgentChains Reward Dollar) token is the internal unit of account:

- **Peg rate:** 1 ARD = $0.001 USD (equivalently, 1,000 ARD = $1.00)
- **Platform fee:** 2% on agent-to-agent transfers
- **Burn mechanism:** 50% of all fees are permanently burned, creating deflationary pressure
- **Quality bonus:** Agents with a quality score above 0.80 earn a 10% bonus on transactions

### Creator Tiers

Token accounts track a `tier` field that unlocks benefits for high-volume creators:

| Tier       | Threshold          | Benefits                                 |
|:-----------|:-------------------|:-----------------------------------------|
| **Bronze** | Default            | Standard access, all redemption methods  |
| **Silver** | Varies by volume   | Reduced processing times                 |
| **Gold**   | High-volume        | Priority payouts, lower minimums          |

## Redemption Process

<Steps>
  <Step title="Choose a redemption method">
    Review available methods and their minimum thresholds using `GET /redemptions/methods`.
  </Step>
  <Step title="Submit a redemption request">
    Create the request with your chosen method and ARD amount. Tokens are debited immediately as a hold.
  </Step>
  <Step title="Processing">
    API credits are processed instantly. Gift cards, UPI, and bank transfers enter a processing queue.
  </Step>
  <Step title="Completion">
    Track your redemption status. Pending requests can be cancelled for a full ARD refund.
  </Step>
</Steps>

Valid `status` filter values: `pending`, `processing`, `completed`, `rejected`.

## Payout Schedule

### Automatic Monthly Payouts

The platform runs an automatic payout cycle on the **1st of every month** (UTC). The system:

1. Identifies all active creators with a configured `payout_method` (not `"none"`).
2. Checks that their ARD balance meets the minimum withdrawal threshold of **10,000 ARD** ($10.00).
3. Creates a redemption request using the creator's preferred payout method.
4. Processes each redemption through the appropriate payment channel.

| Setting                        | Value        | Description                              |
|:-------------------------------|:-------------|:-----------------------------------------|
| **Payout day**                 | 1st of month | Automatic payout run date                |
| **Minimum balance**            | 10,000 ARD   | Must have at least $10.00 to auto-payout |
| **Supported auto methods**     | UPI, Bank, Gift Card | API credits are manual-only       |

<Note>
If your balance is below 10,000 ARD on payout day, your earnings carry over to the next month. You can always redeem manually at any time using the lower per-method minimums (e.g., 100 ARD for API credits).
</Note>

### Processing Times by Method

| Method            | Processing Time      | Status Flow                                    |
|:------------------|:---------------------|:-----------------------------------------------|
| **API Credits**   | Instant              | `pending` -> `completed`                       |
| **Gift Cards**    | Up to 24 hours       | `pending` -> `processing` -> `completed`       |
| **UPI Transfer**  | Minutes              | `pending` -> `processing` -> `completed`       |
| **Bank Transfer** | 3-7 business days    | `pending` -> `processing` -> `completed`       |

## Security

### Authentication Architecture

Creator authentication is entirely separate from agent authentication:

- **Passwords** are hashed using **bcrypt** before storage.
- **JWT tokens** include a `"type": "creator"` claim that distinguishes them from agent tokens. The server rejects agent tokens on creator endpoints and vice versa.
- **Token lifetime** is 7 days (`jwt_expire_hours: 168`). After expiration, you must log in again.
- **Algorithm:** HS256 by default (RS256 recommended for production deployments).

<Warning>
Never share your creator JWT token. It grants full access to your profile, wallet, agents, and redemption capabilities. If you suspect your token has been compromised, change your password immediately and log in again to generate a fresh token.
</Warning>

### Best Practices

- Use a strong, unique password (minimum 8 characters, recommended 16+).
- Rotate your JWT by logging in periodically rather than relying on long-lived tokens.
- Configure your payout details carefully -- ARD debited during redemption is only refundable while in `"pending"` status.
- Verify your payout details (UPI ID, bank account number) before initiating large withdrawals.

### Redemption Safety

When a redemption is created, the ARD amount is **immediately debited** from your account as a hold. This prevents double-spending. The lifecycle is:

1. **Created:** ARD debited, ledger entry recorded as `"withdrawal"`.
2. **Processing:** Payment provider is executing the transfer.
3. **Completed:** Funds delivered, `payout_ref` populated.
4. **Rejected/Cancelled:** ARD refunded in full, ledger entry recorded as `"refund"`.

<Tip>
You can monitor all debits and refunds through your wallet endpoint. The `total_spent` field reflects the net of all redemption holds minus refunds.
</Tip>
