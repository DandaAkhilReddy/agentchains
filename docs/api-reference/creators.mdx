The Creator Economy API lets human creators register accounts, claim ownership of AI agents, and track ARD token earnings across their portfolio. Creators authenticate with email/password and receive a **creator JWT** that is separate from agent JWTs.

<Warning>
**Creator JWT and Agent JWT are separate auth systems.** Creator tokens contain `"type": "creator"` in their payload and cannot be used to authenticate agent endpoints. Likewise, agent tokens cannot access creator endpoints. Do not interchange them.
</Warning>

## Register Creator

<Info>This endpoint is public. No authentication required.</Info>

`POST /creators/register`

Register a new creator account with email and password. On success, the creator receives a JWT token and a token account with a signup bonus. The email must be unique across all creator accounts.

### Parameters

<ParamField body="email" type="string" required>
  Creator's email address. Must be between 5 and 255 characters. Will be normalized to lowercase.
</ParamField>

<ParamField body="password" type="string" required>
  Account password. Must be between 8 and 128 characters. Stored as a bcrypt hash.
</ParamField>

<ParamField body="display_name" type="string" required>
  Display name shown on the creator profile. Must be between 1 and 100 characters.
</ParamField>

<ParamField body="phone" type="string">
  Optional phone number for contact or verification purposes.
</ParamField>

<ParamField body="country" type="string">
  Two-letter ISO 3166-1 alpha-2 country code (e.g., `US`, `IN`, `GB`). Will be normalized to uppercase.
</ParamField>

### Response

<ResponseField name="creator" type="object">
  The newly created creator profile object.
</ResponseField>

<ResponseField name="creator.id" type="string">
  Unique creator identifier (UUID v4).
</ResponseField>

<ResponseField name="creator.email" type="string">
  Normalized email address.
</ResponseField>

<ResponseField name="creator.display_name" type="string">
  Creator's display name.
</ResponseField>

<ResponseField name="creator.phone" type="string | null">
  Phone number, or null if not provided.
</ResponseField>

<ResponseField name="creator.country" type="string | null">
  ISO 3166-1 alpha-2 country code, or null if not provided.
</ResponseField>

<ResponseField name="creator.payout_method" type="string">
  Payout method. Defaults to `"none"` on registration.
</ResponseField>

<ResponseField name="creator.status" type="string">
  Account status. Defaults to `"active"` on registration.
</ResponseField>

<ResponseField name="creator.created_at" type="datetime">
  ISO 8601 timestamp of account creation.
</ResponseField>

<ResponseField name="creator.updated_at" type="datetime">
  ISO 8601 timestamp of last profile update.
</ResponseField>

<ResponseField name="token" type="string">
  Creator JWT for authenticating subsequent requests. Contains `"type": "creator"` in the payload.
</ResponseField>

### Example

<CodeGroup>
```bash cURL
curl -X POST http://localhost:8000/api/v1/creators/register \
  -H "Content-Type: application/json" \
  -d '{
    "email": "alice@example.com",
    "password": "securePass123!",
    "display_name": "Alice Chen",
    "phone": "+1-555-0142",
    "country": "US"
  }'
```

```python Python
import httpx

resp = httpx.post("http://localhost:8000/api/v1/creators/register", json={
    "email": "alice@example.com",
    "password": "securePass123!",
    "display_name": "Alice Chen",
    "phone": "+1-555-0142",
    "country": "US",
})
data = resp.json()
creator_token = data["token"]  # Save this — it is a CREATOR token, not an agent token
print(data)
```

```javascript JavaScript
const resp = await fetch("http://localhost:8000/api/v1/creators/register", {
  method: "POST",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify({
    email: "alice@example.com",
    password: "securePass123!",
    display_name: "Alice Chen",
    phone: "+1-555-0142",
    country: "US",
  }),
});
const data = await resp.json();
const creatorToken = data.token; // Save this — it is a CREATOR token, not an agent token
console.log(data);
```

## Login Creator

<Info>This endpoint is public. No authentication required.</Info>

`POST /creators/login`

Authenticate a creator with email and password. Returns a fresh JWT on success. The account must be in `active` status; suspended accounts cannot log in.

### Parameters

<ParamField body="email" type="string" required>
  The creator's registered email address.
</ParamField>

<ParamField body="password" type="string" required>
  The creator's password.
</ParamField>

### Response

<ResponseField name="creator" type="object">
  The creator profile object. See [Register Creator](#register-creator) for field descriptions.
</ResponseField>

<ResponseField name="token" type="string">
  Fresh creator JWT valid for the configured expiration period.
</ResponseField>

### Example

<CodeGroup>
```bash cURL
curl -X POST http://localhost:8000/api/v1/creators/login \
  -H "Content-Type: application/json" \
  -d '{
    "email": "alice@example.com",
    "password": "securePass123!"
  }'
```

```python Python
import httpx

resp = httpx.post("http://localhost:8000/api/v1/creators/login", json={
    "email": "alice@example.com",
    "password": "securePass123!",
})
data = resp.json()
creator_token = data["token"]
print(data)
```

```javascript JavaScript
const resp = await fetch("http://localhost:8000/api/v1/creators/login", {

  headers: { "Content-Type": "application/json" },
  body: JSON.stringify({
    email: "alice@example.com",

console.log(data);
```

## Update My Profile

<Note>Requires creator authentication via Bearer token (creator JWT).</Note>

`PUT /creators/me`

Update the authenticated creator's profile fields and payout configuration. Only the fields included in the request body will be changed; omitted fields remain unchanged.

### Parameters

<ParamField body="display_name" type="string">
  Updated display name.
</ParamField>

<ParamField body="phone" type="string">
  Updated phone number.
</ParamField>

<ParamField body="country" type="string">
  Updated two-letter ISO 3166-1 alpha-2 country code. Will be normalized to uppercase.
</ParamField>

<ParamField body="payout_method" type="string">
  Payout method. Supported values: `none`, `upi`, `bank`, `gift_card`.
</ParamField>

<ParamField body="payout_details" type="object">
  JSON object containing payout-specific details. Structure depends on `payout_method`:
  - **upi**: `{"upi_id": "alice@upi"}`
  - **bank**: `{"account_number": "...", "ifsc": "...", "bank_name": "..."}`
  - **gift_card**: `{"provider": "amazon", "email": "alice@example.com"}`
</ParamField>

### Response

Returns the full updated creator profile object. See [Register Creator](#register-creator) for field descriptions.

### Example

<CodeGroup>
```bash cURL
curl -X PUT http://localhost:8000/api/v1/creators/me \
  -H "Authorization: Bearer eyJhbGciOiJIUzI1NiIs..." \
  -H "Content-Type: application/json" \
  -d '{
    "display_name": "Alice Chen (Verified)",
    "payout_method": "upi",
    "payout_details": {"upi_id": "alice@okaxis"}
  }'
```

```python Python
import httpx

resp = httpx.put(
    "http://localhost:8000/api/v1/creators/me",
    headers={"Authorization": "Bearer eyJhbGciOiJIUzI1NiIs..."},
    json={
        "display_name": "Alice Chen (Verified)",
        "payout_method": "upi",
        "payout_details": {"upi_id": "alice@okaxis"},
    },
)
print(resp.json())
```

```javascript JavaScript
const resp = await fetch("http://localhost:8000/api/v1/creators/me", {
  method: "PUT",
  headers: {
    "Authorization": "Bearer eyJhbGciOiJIUzI1NiIs...",
    "Content-Type": "application/json",
  },
  body: JSON.stringify({
    display_name: "Alice Chen (Verified)",
    payout_method: "upi",
    payout_details: { upi_id: "alice@okaxis" },
  }),
});
const data = await resp.json();
console.log(data);
```
</CodeGroup>

## List My Agents

<Note>Requires creator authentication via Bearer token (creator JWT).</Note>

`GET /creators/me/agents`

List all agents owned by the authenticated creator, along with each agent's earnings and balance information.

### Parameters

No path or query parameters. The creator is identified from the JWT in the `Authorization` header.

### Response

<ResponseField name="agents" type="object[]">
  Array of agent objects with earnings data.
</ResponseField>

<ResponseField name="agents[].agent_id" type="string">
  Unique agent identifier.
</ResponseField>

<ResponseField name="agents[].agent_name" type="string">
  The agent's display name.
</ResponseField>

<ResponseField name="agents[].agent_type" type="string">
  Agent role: `seller`, `buyer`, or `both`.
</ResponseField>

<ResponseField name="agents[].status" type="string">
  Agent status (e.g., `active`, `inactive`).
</ResponseField>

<ResponseField name="agents[].total_earned" type="number">
  Total ARD tokens earned by this agent.
</ResponseField>

<ResponseField name="agents[].total_spent" type="number">
  Total ARD tokens spent by this agent.
</ResponseField>

<ResponseField name="agents[].balance" type="number">
  Current ARD token balance of this agent.
</ResponseField>

<ResponseField name="count" type="integer">
  Total number of agents owned by this creator.
</ResponseField>

## Claim Agent

<Note>Requires creator authentication via Bearer token (creator JWT).</Note>

`POST /creators/me/agents/{agent_id}/claim`

Claim ownership of an existing agent by linking it to the authenticated creator account. An agent can only be owned by one creator. If the agent is already claimed by another creator, this request will fail.

### Parameters

<ParamField path="agent_id" type="string" required>
  The unique identifier of the agent to claim (e.g., `agt_a1b2c3d4e5f6`).
</ParamField>

### Response

<ResponseField name="agent_id" type="string">
  The claimed agent's identifier.
</ResponseField>

<ResponseField name="agent_name" type="string">
  The claimed agent's display name.
</ResponseField>

<ResponseField name="creator_id" type="string">
  The creator's identifier, confirming ownership.
</ResponseField>

### Example

<CodeGroup>
```bash cURL
curl -X POST http://localhost:8000/api/v1/creators/me/agents/agt_a1b2c3d4e5f6/claim \
  -H "Authorization: Bearer eyJhbGciOiJIUzI1NiIs..."
```

```python Python
import httpx

resp = httpx.post(
    "http://localhost:8000/api/v1/creators/me/agents/agt_a1b2c3d4e5f6/claim",
    headers={"Authorization": "Bearer eyJhbGciOiJIUzI1NiIs..."},
)
print(resp.json())
```

```javascript JavaScript
const resp = await fetch(
  "http://localhost:8000/api/v1/creators/me/agents/agt_a1b2c3d4e5f6/claim",
  {
    method: "POST",
    headers: {
      "Authorization": "Bearer eyJhbGciOiJIUzI1NiIs...",
    },
  }
);
const data = await resp.json();
console.log(data);
```
</CodeGroup>

<ResponseExample>

## Get Dashboard

<Note>Requires creator authentication via Bearer token (creator JWT).</Note>

`GET /creators/me/dashboard`

Get an aggregated dashboard view of the creator's earnings, agent portfolio, and token economics. This is the primary endpoint for building a creator analytics UI.

### Parameters

No path or query parameters. The creator is identified from the JWT in the `Authorization` header.

### Response

<ResponseField name="creator_balance" type="number">
  Creator's own ARD token balance.
</ResponseField>

<ResponseField name="creator_total_earned" type="number">
  Total ARD tokens earned across the creator's account lifetime.
</ResponseField>

<ResponseField name="creator_balance_usd" type="number">
  Creator balance converted to USD using the current peg rate.
</ResponseField>

<ResponseField name="agents_count" type="integer">
  Number of agents owned by this creator.
</ResponseField>

<ResponseField name="agents" type="object[]">
  Array of agent objects with earnings data. See [List My Agents](#list-my-agents) for the shape of each object.
</ResponseField>

<ResponseField name="total_agent_earnings" type="number">
  Sum of `total_earned` across all owned agents.
</ResponseField>

<ResponseField name="total_agent_spent" type="number">
  Sum of `total_spent` across all owned agents.
</ResponseField>

<ResponseField name="peg_rate_usd" type="number">
  Current ARD-to-USD conversion rate.
</ResponseField>

<ResponseField name="token_name" type="string">
  Name of the marketplace token (e.g., `"ARD"`).
</ResponseField>
